  1. 应用层巨头反向收购数据平台。SAP 斥资收购 Dremio 构建 Iceberg 原生湖仓，数据平台格局从"云厂商三国杀"扩展为"应用层 + 平台层"双向整合，ERP 厂商不再满足于对接平台而是直接内化能力。
  2. 实时流与语义湖仓成为 Agent 数据地基标配。IBM Think 2026 发布的 Agentic Enterprise 蓝图中，Confluent 实时流 + watsonx.data 语义湖仓构成 AI 的核心数据基础设施，解决传统湖仓时效性痛点。

总判断：本周发生了 SAP 收购 Dremio 和 IBM Think 2026 两起具有战略意义的事件，共同指向同一趋势——应用层巨头正在将开源湖仓能力内化为自身产品栈的一部分。与此同时，Databricks 通过 Catalog Commits GA 强化了 Unity Catalog 作为跨引擎治理中心的地位。数据平台的竞争维度正从"谁的仓库更快"升级为"谁能成为企业数据的统一协调中心"。Iceberg 作为事实标准表格的地位在本周得到进一步巩固。

Top Signals — 核心信号（3 条）

1. SAP 宣布收购 Dremio，构建 Apache Iceberg 原生企业级湖仓

📎 来源：SAP News Center · PR Newswire · Dremio Official  | 📅 2026年5月4日

SAP 与 Dremio 达成收购协议，将 Dremio 的开源、高性能数据湖仓平台纳入 SAP Business Data Cloud。收购完成后，SAP Business Data Cloud 将转变为 Apache Iceberg 原生的企业级湖仓，统一管理 SAP 数据和非 SAP 数据，无需 ETL 或数据格式转换即可支持联邦查询和实时分析。

Dremio 定位为"专为智能体构建且由智能体管理"的 Iceberg 原生平台，具备无 ETL 联邦查询、AI 语义层、自主集群管理三大特色。交易条款未披露，预计 2026 年 Q3 完成（待监管批准）。

同日，SAP 还宣布收购 AI 表格基础模型公司 Prior Labs，并承诺未来四年投资超 €10 亿+，形成"数据基础设施 + AI 模型"完整组合拳。

🎯 为什么对数据平台重要：这是 ERP/应用层巨头对数据平台领域最具野心的横向扩张之一。SAP 绕过了与 Snowflake/Databricks 的纯合作模式，直接内化湖仓能力。对 Iceberg 生态而言，这是继 Tabular 被 Snowflake 收购后又一重量级背书，进一步确认了 Iceberg 作为开放表格式事实标准的地位。对数据架构师来说，意味着"SAP 数据是否应该进数据湖"这个长期问题有了官方答案——SAP 自己就在建湖。

2. IBM Think 2026：发布"AI 运营模式"蓝图，Confluent 实时流 + watsonx.data 语义湖仓驱动 Agentic Enterprise

📎 来源：IBM via PR Newswire · IBM 中国  | 📅 2026年5月5日

IBM 在波士顿 Think 2026 大会上发布了最全面的企业 AI 和混合云管理扩展方案，定义了 "Agentic Enterprise（智能体企业）"运营模式，由四大集成系统组成：

  • watsonx Orchestrate（下一代）— 多智能体编排控制平面，私密预览
  • Confluent 集成 GA — 基于 Kafka/Flink 的实时数据流正式可用，Confluent Real-Time Context Engine 为 AI 提供实时上下文
  • watsonx.data 升级 — Context Layer 语义层（OpenRAG 开放检索增强生成）+ GPU 加速 Presto 引擎（83% 成本节约 / 30倍性价比，雀巢 PoC）
  • Concert 智能运维平台 — 公开预览；Sovereign Core 主权核心 — GA
🎯 为什么对数据平台重要：IBM 展示了完整的"实时数据流 → 语义化湖仓 → Agent 编排"技术栈。特别是 Confluent + watsonx.data 的 GA 集成，标志着传统数据湖最大的痛点之一——数据新鲜度——正在被系统性地解决。GPU 加速 Presto 的成本数据如果能在更多场景复现，将对湖仓查询经济性产生深远影响。

3. Databricks Catalog Commits 正式发布 (GA)：Unity Catalog 升级为跨引擎事务协调中心

📎 来源：Databricks Release Notes - May 2026  | 📅 2026年5月8日

Databricks 宣布 Catalog Commits 功能正式发布 (GA)——这是一项重大架构更新：

  • Unity Catalog 从元数据管理升级为表的 System of Coordination（协调中心）
  • 支持 多语句、多表事务
  • 覆盖范围：流式表、Delta Sharing、Zerobus、Lakeflow Connect、Mosaic AI Gateway、MLflow、Lakeflow Jobs 触发器
  • Lakeflow Pipelines Editor 同步 GA — Agent 优先体验，集成 Genie Code
🎯 为什么对数据平台重要：Catalog Commits 是 Unity Catalog 自推出以来最具架构意义的升级。它解决了多引擎环境下的核心痛点：如何保证跨引擎事务的一致性和可协调性？Databricks 的答案是让 Unity Catalog 成为协调中枢。这与 SAP 选择 Iceberg、IBM 投资语义层的方向一致——治理和控制平面正在成为数据平台竞争的新焦点。
